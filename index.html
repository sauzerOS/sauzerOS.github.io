Overview

sauzerOS

From scratch distribution based on LFS/BLFS/GLFS using the Hokuto package manager. AI slop website because I'm lazy

Foundation

A lot of packages are based on Arch Linux and the AUR.

Package Management

Hokuto Package Manager

Inspired by KISS package manager, vibe coded from scratch in Go.

Global Configuration

/etc/hokuto.conf
Main configuration file
/etc/hokuto.lock
Pins a package to a specific version (pkgname version)
/etc/hokuto.rebuild
Trigger automatic package rebuilds after build (pgkname pkgtorebuild1 pkgtorebuild2)

Package Structure

pkgdir/

build : bash script executed at build time
checksums : b3sum of source files
depends : package dependencies
files : local files directory
sources : http(s)://url, ftp://url, git+https://github.com/repo #tag or #commit supported, files/file for local files
version : version revision

Optional Configuration (Empty files in pkgdir)

noram
Builds in TMPDIR2
clang
Sets flto=auto for clang builds
asroot
Build package as root
nolto
Disables LTO
nostrip
Disables stripping
interactive
Shows build output and allows interaction with build script

Hokuto Commands

alt <pkg>
List packages with alternatives or show/switch alternatives for a package
bootstrap <dir>
Build a bootstrap rootfs in target directory
build, b (-flag) <pkg>
Build package(s). -a (auto-install), -i (half cpu cores), -ii (one cpu core), --alldeps
cd <pkg>
Change directory to package repository directory
checksum, c <pkg> (-flag)
Fetch sources and generate checksum file for a package. -f (force redownload)
chroot <dir> [cmd]
Enter chroot and run command (default: /bin/bash)
cleanup [options]
Cleanup cache and orphan packages: -sources, -bins, -orphans, -all
edit, e <pkg> (-flag)
Edit a package's build files. -a (edit all files)
find, f <query>
Find which package matches query string
install, i (-flag) <pkg>
Install pre-built packages from the binary cache or a specified .tar.zst file. -force
list, ls <pkg>
List installed packages, optionally filter by name
log
TUI build log viewer
manifest, m <pkg>
Show the file list for an installed package
new, n <pkg>
Create a new package skeleton
uninstall, r <pkg>
Uninstall package(s). -f (force), -y (skip confirmation)
update, u (-flag)
Update repositories and check for upgrades. -i (half cpu), -ii (one cpu)
version, --version
Show hokuto version

build pkgname or pkgname@version

-a
Automatically install the package(s) after successful build.
-alldeps
Force rebuild of all dependencies
-cross=arm64
Enable cross-compilation and store package in /var/cache/hokuto/bin/cross
-generic
Use _GEN flags from /etc/hokuto.conf and store package in /var/cache/hokuto/bin/generic
-i
Use half CPU cores and lowest niceness for build process.
-ii
Use one CPU core and lowest niceness for build process.
-ordered
Force build order based on the target package's depends file.
-r, -rebuilds
Enable post-build actions for dependencies marked with 'rebuild'.
-v, -verbose
Enable verbose output.

depends file options

optional
Package automatically rebuilt once dependency is installed (solves circular deps)
rebuild
Forces rebuild even if installed (requires -rebuilds flag)
make
Only required for build, can be removed via cleanup -orphans
== < > => <= version
e.g. ffmpeg<=7.2.0. Blocks updates or fails build if version mismatch
rust | rustup
Hokuto will prompt which dependency to use

build script environment variables

$1
Build DESTDIR
$2
Package version
$3
Package name
$HOKUTO_ARCH
Build arch (aarch64 or x86_64)
$HOKUTO_CROSS
Cross Mode (1 or 0) and $HOKUTO_CARCH (arm64)
$MULTILIB
Multilib status (1 or 0)
$HOKUTO_ROOT
Root filesystem path
$TMPDIR
Current tmpdir
$HOKUTO_BUILD_DIR
Current build directory

specific variables for cross compilation

Triggered via -cross=arm64 flag.

Default (-cross=arm64)

CC=aarch64-linux-gnu-gcc
CXX=aarch64-linux-gnu-g++
$CROSS_PREFIX=/usr

System (-cross=arm64,system)

CC=aarch64-linux-gnu-gcc
CXX=aarch64-linux-gnu-g++
$CROSS_PREFIX=/usr/aarch64-linux-gnu

Package renamed to aarch64-$pkgname

Simple (-cross=arm64,simple)

CC=cc
CXX=g++
$CROSS_PREFIX=/usr
