hokuto package manager

Package Manager written in Go with minimal optional external dependencies:

  • b3sum
  • git
  • lzip
  • rsync
  • tar
  • wget
  • zstd

The package manager is fully functional without any external dependencies (except for git checkouts) if executed as root.

Usage

hokuto help for basic usage
hokuto function for advanced options

Configuration Files

Configuration files are stored in /etc/hokuto/

  • hokuto.conf: global config file
  • hokuto.lock: pin a package to specific version (pkgname version)
  • hokuto.pkgset: pkgsets for batch updates (hokuto bump -set setname oldver newver)
  • hokuto.rebuild: simple DKMS (triggerpackage package1 package2)
  • hokuto.update: sets package build priority on updates (package1 package2 package3)
  • /etc/hokuto/keys/foo.key: key used to sign packages (use hokuto settings to generate a new key)

Package Structure

  • build: bash file with build instructions
    Available env variables in the script:
    $1: build DESTDIR
    $2: package version
    $3: package name
    $HOKUTO_ARCH: build arch (aarch64 or x86_64)
    $HOKUTO_CROSS: (1 or 0) and $HOKUTO_CARCH (arm64)
    $MULTILIB: (1 or 0)
    $HOKUTO_ROOT: rootdir, usually / but can be set to another dir
    $TMPDIR: current tmpdir
    $HOKUTO_BUILD_DIR: current build dir path
    $HOKUTO_REAL_USER: user that executed hokuto (allows installation in user home directory)
  • checksums: b3sums of source files
  • depends: dependencies
    • pkgname: runtime/build dependency
    • pkgname make: build dependency
    • pkgname optional: package will be automatically rebuilt once the dependency is available (solves circular deps)
    • pkgname rebuild: forces rebuild of package even if already installed, only enabled with -rebuilds build command flag
    • pkgname cross make: cross (arm64) build dependency
    • == <> => <= version (eg ffmpeg<=7.2.0): dependency version constraints. hokuto can build older versions by fetching the package from the repo git history
    • package1 | package2: hokuto will prompt which dependency to use
  • options:
    asroot: build package as root
    binary: binary package with bundled deps, ignored by automatic dependency resolver
    clang: sets flto=auto for clang builds
    cross-simple: use the native host compiler instead of the cross-compiler
    host-tool: don't modify the PATH in the build env to include the cross-toolchain binaries
    interactive: shows build output log and allows interactive scripts
    nocrossopt: don't enable CFLAGS_ARM64 for cross builds
    noram: build in TMPDIR2
    nolto: disable LTO
    nostrip: disable stripping
  • files: local package files
  • post-install: bash script executed after package install
  • .sources: source links with substitutions
    • ${version}: The raw version from the version file.
    • ${version-clean}: Version with underscores (_) replaced by dots (.) (e.g., 1_2_3-1 → 1.2.3-1).
    • ${version_}: Version with dots (.) replaced by underscores (_).
    • ${version-}: Version with dots (.) replaced by dashes (-).
    • ${version-major}: 1.2.3 -> 1
    • ${version-major-minor}: 1.2.3 -> 1.2
    • ${version-sqlite}: custom substitution for sqlite download links
    • ${version-glued}: 1.2.3 -> 123
    • ${version-glued-end}: 1.2.3 -> 1.23
    • ${version-last-only}: 1.2.3.sdk102 -> sdk102
    • ${pkgname}: The name of the package.
  • sources: source links with substitutions replaced with actual links (for repology parsing)
  • version: version revision
